# Kickside UI Host Contract

The host loads module UI from registry declarations. A module does not get a screen by importing it into the host app; it gets a screen by declaring a static bundle and a `view.component` entry.

## Served Bundle

```yaml
- name: ui_fs
  kind: fs.directory
  meta:
    comment: Built module web component bundle.
  directory: ./static

- name: ui_static
  kind: http.static
  meta:
    server: app:gateway
  path: /kickside-example
  fs: kickside.example:ui_fs
```

`directory: ./static` is relative to the module source. The static directory is generated by the module's Vite build and checked in.

## View Component

```yaml
- name: example_view
  kind: registry.entry
  meta:
    type: view.component
    title: Example
    announced: true
    tag_name: kickside-example
    base_path: kickside-example
    entry_point: index.js
    icon: tabler:component
```

- **`tag_name`** — Custom element tag registered by the bundle
- **`base_path`** — Static path without the leading slash
- **`entry_point`** — JS file inside `static/`
- **`title`** — Human-facing title
- **`icon`** — Tabler icon for catalog/navigation surfaces
- **`announced`** — Whether this view is discoverable in the relevant catalog

The host resolves the view, loads `<base_path>/<entry_point>`, then mounts the custom element by `tag_name`.

## Component Kind Links

Component kinds can point at their create/manage views:

```yaml
meta:
  component:
    create:
      view: kickside.example:example_create_view
    manage:
      view: kickside.example:example_manage_view
```

The generic component and Automation screens read these declarations. If a kind needs a custom create or manage UI, declare it here; do not branch on the kind id in the host UI.

## Schema-Owned Detail Components

Some schema fields use a web component picker or detail panel:

```yaml
config_schema:
  thread_id:
    picker: wc-component-picker
    role: primary
    title: Webhook
    detail: kickside-webhook-detail
```

The schema owns the field title, picker, and detail widget. UI code should render the declared field, not infer nouns from ids.

## Host Responsibilities

The host provides:

- authenticated API access through `@wippy-fe/proxy`
- theme and font CSS urls requested by `hostCssKeys`
- custom element loading by tag
- global navigation and realtime delivery
- confirmation, toast, and auth-expired UX

Module UI owns:

- its source and generated static bundle
- its registry view declarations
- its API calls to its module backend
- its tests and local UX states
